# Implementation Plan: Tap-to-Select

**Branch**: `006-tap-to-select` | **Date**: 2026-03-06 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/006-tap-to-select/spec.md`

## Summary

Replace the two `<select>` dropdown elements in the new-match form with a 2-column grid of player name buttons. A 3-step flow (pick P1, pick P2, confirm & start) replaces the dropdown interaction. New ephemeral state variables track pick progress. All e2e test helpers that start matches must update from `selectOption` to tap-based selection.

## Technical Context

**Language/Version**: Vanilla JavaScript ES2022+ (native ES modules)
**Primary Dependencies**: None (production). Vitest 3.x + Playwright (dev only)
**Storage**: No changes — `localStorage` keys unchanged
**Testing**: Vitest 3.x (unit) + Playwright (e2e)
**Target Platform**: Browser (mobile-first, 375px+)
**Project Type**: Web application (static, no build step)
**Performance Goals**: No perceptible delay on pick step transitions
**Constraints**: No frameworks, no bundler; must work with `npx serve`
**Scale/Scope**: Single-user; roster typically 4–8 players
