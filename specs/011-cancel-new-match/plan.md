## Implementation Approach

### liveView.js changes

**1. Cancel button rendering** — add to:
- Pick step 1 HTML (pick grid, no player selected)
- Pick step 2 HTML (pick grid, P1 badge + pick P2)
- Confirm step HTML (alongside Start button)

**2. Event handler: `cancel-new-match`**
```
case 'cancel-new-match':
  _pickStep = null;
  _selectedP1 = null;
  _selectedP2 = null;
  _newMatchExpanded = false;
  refreshNewMatchZone();
```

**3. Event handler: `deselect-player` extension**

Existing: fires only at confirm step.

Extended:
```
case 'deselect-player':
  if (_pickStep === 2) {
    _selectedP1 = null;
    _pickStep = 1;
    refreshNewMatchForm(); // reads _pickStep to render correct step
  } else if (_pickStep === 'confirm') {
    _selectedP1 = null;
    _selectedP2 = null;
    _selectedTarget = 7; // clear target (reset to default)
    _pickStep = 1;
    refreshNewMatchForm(); // reads _pickStep to render correct step
  }
```

**4. Deselect affordance at step 2** — render a deselect button/icon on the P1 badge when `_pickStep === 2`.
