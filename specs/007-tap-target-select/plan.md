# Implementation Plan: Tap-to-Select Target Score Grid

**Branch**: `007-tap-target-select` | **Date**: 2026-03-06 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/007-tap-target-select/spec.md`

## Summary

Replace the `<input type="number">` target field in the new-match confirmation step with a 10-button tap grid showing odd values 3–21. A single new ephemeral state variable (`_selectedTarget = 7`) tracks the selection. All e2e test helpers that currently `fill()` the target input must be updated to use `pick-target` button clicks (or omit the step when relying on the default 7).

## Technical Context

**Language/Version**: Vanilla JavaScript ES2022+ (native ES modules)
**Primary Dependencies**: None (production). Vitest 3.x + Playwright (dev only)
**Storage**: No changes — `localStorage` keys unchanged
**Testing**: Vitest 3.x (unit) + Playwright (e2e) via Playwright MCP
**Target Platform**: Browser (mobile-first, 375px+)
**Project Type**: Web application (static, no build step)
**Performance Goals**: No perceptible delay on button selection
**Constraints**: No frameworks, no bundler; must work with `npx serve`
**Scale/Scope**: Single-user; 10 preset target values
