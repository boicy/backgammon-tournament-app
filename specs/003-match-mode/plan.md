# Implementation Plan: Match-Mode Tournament Nights

**Branch**: `003-match-mode` | **Date**: 2026-03-05 | **Spec**: [spec.md](spec.md)

## Summary

Replace the current flat game-recording model with a match-first system. A tournament night consists of multiple independent matches (Player A vs Player B, playing to a target score). Games are recorded within a match using the existing cube/gammon logic. Matches auto-complete when the target is reached. Multiple matches can run simultaneously. The night leaderboard ranks by match wins then total points. Legacy archives remain readable.

## Technical Context

**Language/Version**: Vanilla JavaScript ES2022+, HTML5, CSS3
**Primary Dependencies**: None (production); Vitest 3.x + Playwright (dev)
**Storage**: `localStorage` — five existing keys; one new key `backgammon:matches`
**Testing**: Vitest (unit), Playwright via MCP (e2e)
**Target Platform**: Browser (static, any CDN host)
**Project Type**: Single-page web application (no bundler)
**Performance Goals**: Match hub renders < 100ms with up to 10 active matches
**Constraints**: Offline-capable, no server, mobile-first (375px+)
**Scale/Scope**: Single club (~20 players, ~10 simultaneous matches per night)
