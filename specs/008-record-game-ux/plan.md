# Implementation Plan: Record Game UX — Winner Tap-Select & Prominent Save

**Branch**: `008-record-game-ux` | **Date**: 2026-03-07 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/008-record-game-ux/spec.md`

## Summary

Replace the winner `<select>` dropdown in the inline game-recording form with two large tap-target buttons (one per player), using the `.pick-btn` / `.pick-btn--selected` pattern already established in features 006/007. Simultaneously make the Save/Submit button a full-width, 48px-tall primary CTA. No new dependencies, no model changes — all work is confined to `liveView.js`, `styles.css`, and test files.

## Technical Context

**Language/Version**: Vanilla JavaScript ES2022+ (native ES modules), HTML5, CSS3
**Primary Dependencies**: None (production); Vitest 3.x + Playwright (dev)
**Storage**: localStorage — no changes; `_selectedWinner` is ephemeral module-level state
**Testing**: Vitest (unit/DOM) + Playwright via Playwright MCP (e2e)
**Target Platform**: Static browser app; primary use on smartphone
**Project Type**: Single-page web application
**Performance Goals**: Interaction response <100ms (DOM toggle, no async)
**Constraints**: No frameworks, no bundler; offline-capable static deployment
**Scale/Scope**: Single liveView.js module; ~2 new CSS rules; 6 existing test files updated
